定时规则

当前底层 schedule: 每 30 分钟检查一次 实际自动同步: 由 AUTO_RUN_TIMES 和 RUN_WINDOW_MINUTES 决定 说明: GitHub Actions 的 cron 使用 UTC，网页里填写的是北京时间。自动定时是否执行由 AUTO_RUN_ENABLED 控制。

规则与说明

使用规则

  • 用途是个人可访问性测试和同步功能验证。
  • 手动同步会立即触发 GitHub Actions，并使用输入的步数。
  • 今日累计模式会按当天日期和北京时间生成累计值。
  • 自动定时停用后，手动同步仍然可用。

配置说明

  • USER 和 PWD 存在 Secrets；页面只能覆盖，不能读取明文。
  • AUTO_RUN_ENABLED、AUTO_RUN_TIMES、RUN_WINDOW_MINUTES、STEP_RANGE、OPEN_GET_WEATHER、AREA 存在 Variables，可读取和修改。
  • 多账号格式：账号和密码都用 # 分隔，并保持数量一致。
  • 天气关闭时，AREA 建议为 NO。

安全提示

  • GitHub token 不要长期复用，建议专门为控制台创建并定期撤销。
  • 公开仓库不要把账号、密码或 token 写入文件。
  • 浏览器保存 token 后，只适合在你信任的个人设备上使用。

故障排查

  • 如果手动同步失败，先刷新状态，确认 workflow 是 active。
  • 如果变量保存失败，检查 token 是否具备仓库变量写权限。
  • 如果 Actions 成功但第三方未同步，检查小米运动/Zepp 绑定关系。
